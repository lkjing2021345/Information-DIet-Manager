# JSON输入标准

--- 

## 1. 标准请求结构
```json src/lsj/src/main_input_standard.md
{
  "records": [
    {
      "id": "1",
      "title": "今天心情很好，工作很顺利",
      "url": "https://example.com/news/1",
      "timestamp": "2025-03-08 10:30:00"
    }
  ],
  "options": {
    "include_emotions": false,
    "batch_size": 128,
    "detailed": true
  }
}
```

---

## 2. 必填字段
- `records`：数组，不能为空
- `records[].title`：字符串，不能为空

---

## 3. 推荐字段
- `records[].id`：业务主键
- `records[].url`：页面 URL，便于分类
- `records[].timestamp`：推荐时间字段
- `records[].visit_time`：可替代 `timestamp`
- `records[].ts`：Unix 时间戳，秒或毫秒均可

---

## 4. options 说明
- `include_emotions`：是否返回情绪分析结果，默认 `false`
- `batch_size`：批处理大小，默认 `128`
- `detailed`：是否返回详细评估结果，默认 `false`

---

# CSV 输入标准

---

## 1. 必填列
| 列名    | 类型     | 说明             |
|-------|--------|----------------|
| title | string | 页面标题或文本内容，不能为空 |

---

## 2. 推荐列
| 列名         | 类型            | 说明                             |
|------------|---------------|--------------------------------|
| id         | string        | 业务主键，建议提供                      |
| url        | string        | 页面 URL，便于分类                    |
| timestamp  | string        | 推荐时间字段，如 `2025-03-08 10:30:00` |
| visit_time | string        | 可替代 `timestamp`                |
| ts         | number/string | Unix 时间戳，秒或毫秒均可                |

---

## 3. CSV 表头示例
```csv src/lsj/src/main_input_standard.csv.md
id,title,url,timestamp
1,今天心情很好，工作很顺利,https://example.com/news/1,2025-03-08 10:30:00
2,Python 教程入门,https://example.com/learn/python,2025-03-08 21:00:00
```
